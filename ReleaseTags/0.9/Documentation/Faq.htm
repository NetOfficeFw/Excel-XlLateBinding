Frage: Meine Programmiersprache unterstützt keine Properties mit Parametern ?
Antwort: Die syntaktischen Workarounds sind mit den Primary Interop Assemblies identisch.
Nutzen Sie ein Property mit Parameter als Methode via get_Value(param).


Frage: Meine Sprache unterstützt keine optionalen Parameter ?
Antwort: Für optionale Parameter steht jeweils eine minimale und eine maximale Funktionsüberladung
zur Verfügung. Für nicht verwendete Parameter nutzen Sie System.Reflection.Missing.Value.


Frage: Habe ich Zugriff auf die nativen COM Proxies?
Antwort: Ja, jedes Objekt implementiert das Property UnderlyingObject.


Frage: Ich arbeite mit verschiedensprachigen Excel Versionen. Wie kann ich die ThreadCulture ändern ?
Antwort: Ändern Sie die Kultur nach Wunsch: LateBindingApi.Core.Settings.ThreadCulture
Die Vorseinstellung ist "en-us"


Frage: Kann ich Events nutzen ?
Antwort: Ja, ein Beispiel dazu finden Sie in Example08.csproj


Frage: Welchen Workaround gibt es für den Variant Datentyp ?
Antwort: Lesen Sie dazu bitte COM Variant & COMObject.htm
